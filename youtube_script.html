Part 1: Base Paper Explanation (0:00 - 5:00)

[Visual on Screen] Open the DarkIR_Official_Paper.pdf to the Title and Abstract.

Speaker (Kumar, Aditya, or Roshan):

"Hello everyone, welcome to our Deep Learning course project presentation. Today we are presenting our work on DarkIR: Robust Low-Light Image Restoration. This paper was published very recently at CVPR.

The core problem that this paper tackles is that when you take a photo in severely low-light environments, you don't just get a dark image—you get a coupled degradation of darkness, blur from shaky camera hands, and heavy sensor noise."

[Visual on Screen] Scroll down to the Introduction block and Highlight Equation 1 ($y = \gamma(x \otimes k) + n$).

Speaker:

"To understand the complexity, look at this formulation. Traditional methods try to fix these sequentially—first denoise, then brighten, then deblur. But as seen in Equation 1, the clean image $x$ is simultaneously blurred by $k$, altered by the camera response function $\gamma$, and corrupted by noise $n$. Trying to fix one without the others leads to artifacts. DarkIR proposes to solve them together using a dual-domain network."

[Visual on Screen] Scroll down to the Architecture Diagram (Figure 2) and highlight the two distinct paths (Spatial and Frequency).

Speaker:

"The base DarkIR model avoids computationally expensive Transformers. Instead, it relies on an efficient CNN with two primary processing streams: 1. A Spatial Domain Path using dilated convolutions to fix local corruptions like noise and blur. 2. A Frequency Domain Path using Fast Fourier Transforms (FFTs) to capture global illumination and structural color adjustments. This makes the baseline incredibly fast and lightweight. However, during our reproduction, we found several areas where we could push the performance and stability even further."

Part 2: Our Novel Contributions (5:00 - 10:00)

[Visual on Screen] Switch to your PPT presentation (specifically the Contributions flow diagram we generated in LaTeX).

Speaker:

"While reproducing this architecture, we implemented four distinct novel contributions to improve visual quality and model learning."

Contribution 1: Phase-Aware Frequency Modulation

"Our first contribution is in the Frequency Path. The standard DarkIR primarily enhances the magnitude of the spectrum and leaves the phase mostly untouched. However, phase contains critical structural information. We introduced a completely new learnable phase-weighting branch." [Visual on Screen] Show a quick code snippet or mention: 

# Our Code Addition:
phase = torch.atan2(imag, real + 1e-8)
phase_weight = torch.sigmoid(self.phase_conv(phase))

"By explicitly mapping torch.atan2 inside the custom layer, the network intentionally shifts phase geometry, leading to crisper edges."

Contribution 2: Content-Adaptive Receptive Fields

"Our second contribution targets the Spatial Path. The base paper hardcodes dilated convolutions. We implemented a dynamic spatial block using Global Pooling and Softmax." [Visual on Screen] Show snippet:

# Our Code Addition:
weights = F.softmax(self.routing_mlp(global_pool(x)), dim=1)
out = w1*branch1 + w2*branch2 + w3*branch3

"Now, the model actively 'votes' via F.softmax on whether it needs to rely on local $3 \times 3$ patterns for noise, or wide $9 \times 9$ patterns to repair blurs, adapting on a per-image basis."

Contribution 3: Gated Channel Attention Matrix

"Third, we upgraded the feature gating mechanism from a simple element-wise multiplication to channel chunk-routing." [Visual on Screen] Show snippet:

# Our Code Addition:
x1, x2 = torch.chunk(x, 2, dim=1)
gate = torch.sigmoid(self.attention_net(x1))
out = torch.cat([x1, x2 * gate], dim=1)

"We split the channels using torch.chunk, learn an attention gate from the first half, and multiply it by the second. This selectively filters out noisy channels before reconstruction."

Contribution 4: Dual-Spectrum Frequency Loss

"Finally, we proposed a Novel Frequency Loss. Traditional training only penalizes L1 combinations at the pixel level. We wrote a custom loss function that enforces L1 consistency explicitly on the Amplitude and Phase maps." [Visual on Screen] Show snippet:

# Our Code Addition:
loss_freq = target_l1(mag_pred, mag_gt) + 0.1 * target_l1(phase_pred, phase_gt)

"By adding a specific 0.1 * phase_l1 weight, we forced the network to respect frequency structures. We also had to mathematically bound the phase mappings strictly to prevent NaNs on our GPU."
